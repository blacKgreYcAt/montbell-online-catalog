## 資料欄位說明

| 欄位順序 | 欄位名稱 (中文) | 欄位代碼 | 資料型態 | 必填 | 說明 |
|---------|----------------|---------|---------|------|------|
| A | 改款/推薦 | NEW_FLAG | 文字 | ✗ | 標記商品狀態 (27SS、推薦、空值) |
| B | 系列 | SERIES | 文字 | ✓ | 商品分類/系列名稱 |
| C | 型號 | MODEL_NUMBER | 數字 | ✓ | 7 位數商品型號 |
| D | 品名 | NAME | 文字 | ✓ | 商品英文名稱 |
| E | 色號名稱 | COLOR_CODES | 文字 | ✓ | 顏色代碼 (空格分隔) |
| F | 尺寸 | SIZES | 文字 | ○ | 尺寸列表 (空格分隔) |
| G | 訂價 | PRICE | 數字/公式 | ✓ | 商品價格 (NT$) 或公式 |
| H | 特色 | DESCRIPTION | 文字 | ○ | 商品特色/描述 (含換行符) |
| I | 頁碼 | PAGE_NUMBER | 數字 | ○ | 原始目錄頁碼 |

## 資料驗證規則

### 必填欄位驗證
```
商品必須同時具備以下欄位才能轉換：
- 型號 (C 欄) - 不可為空
- 品名 (D 欄) - 不可為空
- 訂價 (G 欄) - 必須是數字，不可是公式
```

### 跳過規則
- ❌ 型號或品名為空 → 跳過該筆
- ❌ 訂價為公式 (=VLOOKUP...) → 跳過該筆
- ❌ 訂價不是有效的數字 → 跳過該筆

### 範例：轉換統計
```
原始資料列：1,154 列 (扣掉標題)
+ 成功轉換：1,152 件
+ 跳過（公式）：1 件
+ 其他錯誤：1 件
```

## 型號格式化

### 格式化規則
```
輸入：任意數字 (3-7 位)
輸出：7 位數字（左填零）

範例：
  1128573  → 1128573 ✓
  128573   → 0128573 ✓
  28573    → 0028573 ✓
```

### 商品 ID 生成
```
商品 ID = "prod-" + 7位型號

範例：
  型號 1128573
  → ID: prod-1128573
```

## 顏色代碼處理

### 顏色代碼格式
- **儲存格式：** 多個代碼以空格分隔 (E 欄)
- **範例：** "BK NV" (黑色和深藍色)

### 顏色代碼清單 (共 247 個)

#### 基本顏色 (單字母或簡寫)
```
BK   = Black (黑)
WH   = White (白)
RD   = Red (紅)
BL   = Blue (藍)
GN   = Green (綠)
YL   = Yellow (黃)
GY   = Grey (灰)
BR   = Brown (棕)
CH   = Charcoal (深灰)
NV   = Navy (深藍)
```

#### 複合顏色 (兩色組合)
```
BK/CH = Black/Charcoal
BK/GY = Black/Grey
IV/DN = Ivory/Denim
WT/RD = White/Red
等等...
```

#### 數字變體 (大小號變異)
```
BK23, BK24, BK25 等
NV22, NV23, NV24 等
```

#### 描述顏色 (特殊名稱)
```
CAMO    = Camouflage (迷彩)
FOG     = Fog (霧)
PANSY   = Pansy (紫羅蘭)
RUST    = Rust (鐵鏽)
THYM    = Thyme (百里香)
等等...
```

#### 佔位符
```
#1, #2, #3 ... #21 = 產品套色或多色商品
```

### 顏色代碼統計
```
總顏色代碼：246 個
顏色組合：1,086 種
最常見的顏色代碼：BK (黑色)
```

## 尺寸處理

### 尺寸格式
- **儲存格式：** 多個尺寸以空格分隔 (F 欄)
- **範例：** "XS S M L XL"

### 常見尺寸類型

#### 衣服尺寸
```
XS, S, M, L, XL, XXL 等
```

#### 褲子尺寸 (含短版)
```
M-S (M尺寸短版)
L-S (L尺寸短版)
XL-S (XL尺寸短版)
M-L (M尺寸長版)
L-L (L尺寸長版)
XL-L (XL尺寸長版)
```

#### 其他尺寸
```
ONE SIZE (單一尺寸)
FREE (自由尺寸)
F (自由尺寸簡寫)
```

### 處理規則
```
1. 讀取 F 欄的尺寸文字
2. 按空格分隔成陣列
3. 去除首尾空白
4. 儲存到 JSON 的 sizes 陣列
```

## 常見問題

### Q1: 如何更新商品資料？

**A:** 遵循以下步驟：

1. **編輯 Excel 檔案**
   ```
   檔案：27SS 正式_ 工作本連結資料27SS 資料檔.xlsx
   工作表：公司選品連結用資料
   ```

2. **執行轉換腳本**
   ```bash
   cd "{工作資料夾}"
   /c/Python314/python read_excel.py
   ```

3. **驗證輸出**
   - 檢查轉換成功的商品數
   - 確認沒有錯誤訊息
   - 檢查 products.json 是否已更新

4. **複製到公開目錄**
   ```bash
   cp products.json public/products.json
   ```

5. **提交並部署**
   ```bash
   git add public/products.json
   git commit -m "Update: Refresh product data from Excel"
   vercel --prod
   ```

### Q2: 如何新增商品？

**A:** 在 Excel 中新增一列，填入：
- 型號 (C 欄) - 必填
- 品名 (D 欄) - 必填
- 系列 (B 欄) - 必填
- 顏色代碼 (E 欄) - 必填 (至少一個)
- 訂價 (G 欄) - 必填 (數字，不可是公式)
- 其他欄位可選

### Q3: 為什麼某些商品沒有被轉換？

**A:** 常見原因：
- 型號或品名為空 → 補充缺失資訊
- 訂價是公式 → 改為具體數字
- 訂價為非數字 → 確認格式正確

### Q4: 商品顏色代碼如何確定？

**A:** 
- 查看官方 Montbell 網站的色號
- 使用英文簡寫代碼 (BK, RD, NV 等)
- 多色用空格分隔 (BK NV)
- 不確定時保留為原始代碼

### Q5: 型號可以有少於 7 位嗎？

**A:** 可以，腳本會自動補零
- 型號 1573 → 自動轉換為 0001573
- 但建議在 Excel 中統一為 7 位格式

### Q6: 如何刪除商品？

**A:** 
- **方法1：** 在 Excel 中刪除該列，重新轉換
- **方法2：** 直接編輯 products.json，刪除該件商品

### Q7: 尺寸如何新增？

**A:** 在 F 欄新增尺寸，以空格分隔
```
範例：XS S M L XL XXL
```

### Q8: 如何修改商品標籤？

**A:** 編輯 A 欄 (改款/推薦)
- 27SS → 標籤為 "27SS 新品"
- 推薦 → 標籤為 "推薦"
- 空值 → 沒有標籤

## 🤖 自動化處理 (新增)

### 自動化 Agent 功能

**Montbell Product Data Processor Agent** 可以自動化完整的商品資料處理流程：

#### 功能範圍
1. **自動季節檢測**
   - 從 Excel 文件名自動識別季節 (SS 或 FW)
   - 為所有商品自動添加 season 字段
   
2. **數據轉換**
   - 執行 read_excel.py 腳本
   - 驗證所有必填欄位
   - 格式化顏色代碼（轉換為小寫）
   - 處理尺寸和價格數據

3. **JSON 生成**
   - 生成 public/products.json
   - 驗證 JSON 結構完整性
   - 統計商品分類和顏色數

4. **版本控制**
   - 自動 git 提交
   - 生成包含統計信息的提交信息
   - 推送到 GitHub

5. **部署**
   - 部署到 Vercel 生產環境
   - 驗證部署成功
   - 測試 API 端點

6. **驗證報告**
   - 確認所有商品包含 season 字段
   - 驗證季節過濾正常運作
   - 生成完整的部署報告

#### 使用方式
