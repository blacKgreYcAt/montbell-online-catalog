# 版本控制指南

## 📌 版本管理策略

此專案採用**語義化版本控制** (Semantic Versioning)，格式：`v主版本.次版本.修訂版本`

### 版本標籤規則

- **v0.1.0** - 初始化專案結構
- **v0.2.x** - 核心功能開發
- **v0.3.x** - 功能完善和優化
- **v1.0.0** - 首個生產環境版本

### 提交流程

每個穩定的開發階段完成後：

```bash
# 1. 檢查當前狀態
git status

# 2. 查看變更
git diff

# 3. 提交變更
git add .
git commit -m "描述此階段的更改"

# 4. 標記版本（穩定版本）
git tag -a v0.X.X -m "版本說明"

# 5. 查看所有版本
git tag -l
```

## 🔄 快速回退

當發現問題時，可以快速回到上一個正常的版本：

```bash
# 查看所有標籤和提交歷史
git log --oneline --graph --all --decorate

# 回到特定版本
git checkout v0.X.X

# 或建立新分支從該版本繼續開發
git checkout -b bugfix/issue-name v0.X.X
```
